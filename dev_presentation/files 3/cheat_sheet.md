# Cheat sheet — что значит каждый термин и метрика

Шпаргалка на случай вопросов. Что есть что в твоей презентации.

---

## МОДЕЛИ И ИНСТРУМЕНТЫ

**VITS** — твоя основная модель (Variational Inference with adversarial learning for end-to-end TTS). Превращает текст в звук одной моделью, без отдельного вокодера. Внутри: VAE + normalizing flow + GAN-декодер + предсказатель длительности.

**vits-ljs** — готовый предобученный чекпоинт VITS (от kakao-enterprise), обучен на LJSpeech. Твоя база для finetune.

**RVC** (Retrieval-based Voice Conversion) — инструмент, который меняет ТЕМБР голоса в аудио, но сохраняет произношение/слова. Ты прогнал им LJSpeech, чтобы получить дроид-голос.

**LJSpeech** — стандартный английский TTS-датасет, 13,100 коротких аудио с текстом. Исходные данные.

**LJSpeech-B1** (`Dmi1tr13/ljspeech-b1`) — ТВОЙ датасет: LJSpeech, прогнанный через RVC в голос B1. Лежит на Hugging Face.

**HF / Hugging Face** — платформа для моделей и датасетов. Там твой датасет и веса.

**MAS** (Monotonic Alignment Search) — алгоритм выравнивания: какой текстовый токен какому аудио-кадру соответствует. Динамическое программирование (как Viterbi). Ты взял ГОТОВУЮ референсную реализацию (честно, не с нуля).

## ЧАСТИ МОДЕЛИ

**Generator (генератор)** — основная часть: текст → waveform (звук). Состоит из text encoder, flow, duration predictor, decoder.

**Discriminator (дискриминатор)** — отличает настоящее аудио от сгенерированного. Заставляет генератор делать реалистичный звук. ТЫ написал с нуля (HF даёт VITS только для inference, дискриминатора там нет).

**MPD** (Multi-Period Discriminator) — часть дискриминатора. Режет 1D-аудио в 2D по ПЕРИОДАМ (простые числа 2,7,13...137). Ловит высоту тона и гармоники.

**MSD** (Multi-Scale Discriminator) — вторая часть. Смотрит аудио на 3 МАСШТАБАХ (через avg-pool). Ловит структуру на разных временных масштабах.

**Posterior encoder** — кодирует реальное аудио в латент `z` (только на обучении).

**Flow (normalizing flow)** — обратимое преобразование `z → z_p`, связывает аудио-латент с текстовым prior.

**Decoder** — превращает латент в waveform (звук). Это HiFi-GAN-стиль.

## ЛОССЫ (функции потерь) — что меряют

**recon (reconstruction, L1 mel)** — насколько спектрограмма сгенерированного звука близка к реальной. ГЛАВНЫЙ лосс, вес ×45. Минимум 0.404 на шаге 26,700.

**KL (KL divergence)** — насколько текстовый prior совпал с аудио-posterior. Показывает, что выравнивание работает. Минимум 1.52.

**duration (duration predictor loss)** — насколько верно модель предсказывает длительность каждого токена (ритм речи). Был сломан (100-475), починили → ~1.6. Минимум 1.55.

**adversarial (adv)** — состязательный лосс генератора: насколько хорошо обманывает дискриминатор. ~4 = здоровый баланс. Минимум 3.4.

**d (discriminator loss)** — лосс дискриминатора. ~5.5 стабильно = ни одна сторона не побеждает.

**fm (feature-matching)** — сравнивает внутренние признаки (feature maps) дискриминатора для реального и фейка. Стабилизирует GAN. ~2.

**g (generator total)** — суммарный лосс генератора (все 5 вместе). ~28, доминирует recon×45.

## МЕТРИКИ ОЦЕНКИ (это часть Ильи, но знай)

**WER** (Word Error Rate) — доля ошибочных слов. Whisper расшифровывает твой звук, сравнивают с текстом. Меньше = разборчивее.

**CER** (Character Error Rate) — то же по символам.

**MCD** (Mel Cepstral Distortion) — спектральное расстояние до эталонного дроид-голоса, в дБ. Меньше = ближе к дроиду.

**F0 RMSE** — расстояние по высоте тона (pitch) до эталона. Просодия.

Главный результат (если спросят): MCD упал 10.5 → 6.2 (голос стал дроидом), но WER вырос 9 → 29% (разборчивость упала). Классический размен тембр-vs-разборчивость.

## ОБУЧЕНИЕ — технические термины

**AMP** (Automatic Mixed Precision) — обучение в fp16 вместо fp32. ЭКОНОМИТ память (примерно вдвое) → влезает больший батч. STFT и MAS оставлены в fp32 (чувствительны).

**gradient accumulation** — копим градиенты с нескольких маленьких батчей, потом делаем шаг. Даёт большой ЭФФЕКТИВНЫЙ батч при маленьком физическом (для 8GB). НЕ "accommodation"!

**grad clip (gradient clipping)** — обрезка нормы градиента, чтоб не было взрывов. Было 1.0 (душило генератор), стало 10.0.

**disc warmup** — обучаем дискриминатор отдельно первые 1500 шагов, прежде чем он влияет на генератор. Защищает предобученный декодер.

**LR (learning rate)** — скорость обучения. 1e-4 (мягче, чем 2e-4 для обучения с нуля — это finetune).

**segment_size 8192** — сколько сэмплов waveform режем в декодер за раз. Влезает в 8GB.

**batch_size** — сколько примеров за раз. У тебя физический 1 + accumulation = больший эффективный.

**num_workers=0** — без параллельной загрузки данных (на Windows DataLoader падал с error 1455).

**epoch (эпоха)** — один полный проход по датасету. У тебя 5 эпох, ~1.9ч каждая = 9.68 GPU-часов.

**GPU-hours** — часы работы видеокарты. 9.68 (финальный прогон) + ~3× на неудачные + RVC.

**checkpoint** — сохранённое состояние модели на каком-то шаге. Ты выбираешь не последний, а у минимума recon (~26,700) — дальше GAN деградирует.

**step (шаг)** — одно обновление весов. Всего ~62,000 шагов за 5 эпох.

## ТРИ ОШИБКИ — НЕ ПЕРЕПУТАЙ НА ЗАЩИТЕ

1. **Trade-off:** качество модели НЕ может быть ЛУЧШЕ учителя (RVC = потолок). НЕ говори "better".
2. **AMP** ЭКОНОМИТ память (а не "увеличивает батч" напрямую). Больший батч — следствие.
3. **5 шагов** в train loop: LOAD → forward → loss → backward → UPDATE. Не забудь LOAD и UPDATE.

## СЛОВА, КОТОРЫЕ ПУТАЛ
- **weights** (веса), не "rights"
- **accumulation** (накопление), не "accommodation"
- **generator**, не "neurota"
- **phonemes** (фонемы), не "names/votes"
- **adversarial** или просто "competition"

## ВОЗМОЖНЫЕ ВОПРОСЫ — короткие ответы

**Почему простые числа в периодах?** — чтобы периоды не имели общих делителей, ловят разные гармоники без перекрытия.

**Почему recon весит 45, а не 1?** — спектр (как звучит) важнее всего; GAN-лоссы только добавляют реализм.

**Почему не последний чекпоинт?** — recon достигает минимума ~26,700, дальше GAN медленно деградирует. Беру у минимума, по слуху.

**Почему MAS не с нуля?** — это поисковый алгоритм (динамическое программирование), стандартный, нет смысла переписывать. Взял референс, честно.

**Почему WER вырос после finetune?** — разменял разборчивость на тембр дроида. Учитель (RVC) сам не идеален → потолок качества ограничен.

**Что бы улучшил?** — заморозить text/timing front-end (стабильнее длительности → ниже WER), + добавить акцент.
